# Avances 28/03/2026 - Implementación Core NX-8 Panel Simulator

## Resumen

Implementación completa del simulador NX-8 desde cero en TypeScript (Batches 2-13).

## Cambios Realizados

### Batch 2: Config + Logger
- `src/config/env.ts` — Validación de variables de entorno con Zod
- `src/config/defaults.ts` — Constantes y valores por defecto
- `src/observability/logger.ts` — Logger pino con pino-pretty

### Batch 3: Motor de Estados
- `src/core/state/types.ts` — Tipos TypeScript para Panel, Partición, Zona
- `src/core/state/panel.ts` — PanelStateManager (online, AC, batería, tamper, trouble)
- `src/core/state/partition.ts` — PartitionStateManager (armar/desarmar, alarma, delays)
- `src/core/state/zone.ts` — ZoneStateManager (abrir/cerrar, alarma, bypass, tamper, fault)

### Batch 4: Sistema de Eventos
- `src/core/events/types.ts` — Tipos de eventos (26 tipos clasificados)
- `src/core/events/eventBus.ts` — SimEventBus basado en EventEmitter
- `src/core/events/eventHistory.ts` — Buffer circular con paginación
- `src/core/events/eventGenerator.ts` — Coordinador de estado+eventos

### Batch 5: Protocolo ASCII NX-584
- `src/protocols/nx584/types.ts` — Interfaces del protocolo
- `src/protocols/nx584/ascii/messageTypes.ts` — Catálogo de mensajes
- `src/protocols/nx584/ascii/framing.ts` — Construccion/parseo de tramas con checksum XOR
- `src/protocols/nx584/ascii/encoder.ts` — AsciiEncoder (estado -> trama)
- `src/protocols/nx584/ascii/decoder.ts` — AsciiDecoder (trama -> mensaje)

### Batch 6: Servidor TCP
- `src/server/tcp/clientManager.ts` — Gestión de clientes TCP
- `src/server/tcp/tcpServer.ts` — Servidor TCP con handler de requests/commands
- `src/server/tcp/broadcaster.ts` — Broadcasting de eventos a clientes

### Batch 7: Lab Mode
- `src/lab/types.ts` — Configuración de condiciones de laboratorio
- `src/lab/labMode.ts` — Inyección de delays, jitter, corrupción y silencios

### Batch 8: Observabilidad
- `src/observability/metrics.ts` — Métricas internas del simulador
- `src/observability/health.ts` — Endpoints /health y /ready

### Batch 9: API Fastify
- `src/control-api/schemas/` — Validación Zod para todos los endpoints
- `src/control-api/routes/panel.ts` — Estado completo del panel
- `src/control-api/routes/zones.ts` — CRUD de zonas
- `src/control-api/routes/partitions.ts` — Armar/desarmar particiones
- `src/control-api/routes/events.ts` — Histórico de eventos
- `src/control-api/routes/faults.ts` — Inyección de fallas
- `src/control-api/routes/sim.ts` — Control del simulador
- `src/control-api/routes/scenarios.ts` — Carga/ejecución de escenarios
- `src/control-api/routes/lab.ts` — Condiciones de laboratorio
- `src/control-api/server.ts` — Servidor Fastify principal

### Batch 10: Escenarios
- `src/core/scenarios/types.ts` — Tipos de configuración de escenarios
- `src/core/scenarios/scenarioLoader.ts` — Carga YAML/JSON
- `src/core/scenarios/scenarioRunner.ts` — Motor de ejecución
- 5 escenarios ejemplo en `scenarios/`

### Batch 11: Entry Point
- `src/index.ts` — Orquestador principal con shutdown graceful

### Batch 13: Tests
- 8 suites de tests unitarios (70 tests)
- Cobertura de panel, particiones, zonas, eventos, encoder, decoder, framing, lab mode
